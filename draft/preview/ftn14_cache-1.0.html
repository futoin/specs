1. Concept

This spec is trying to be as simple as possible to meet very basic needs of caching.

3. Native interface

  • Extend "futoin.cache"
  • Functions:
    • void getOrSet( as, key_prefix, params, callable, ttl )
      • key_prefix - unique key prefix
      • params - parameters to be passed to callable
      • callable( params.. ) - a callable which is called to generated value on cache miss
      • tt - time to live to use, if value is set on cache miss
      • NOTE: the actual cache key is formed with concatenation of key_prefix and join of params values
      • NOTE: implementation should mitigate cache hammering to achieve single execution of callable per cache miss
